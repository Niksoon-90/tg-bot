# Размещение Telegram бота — бесплатные варианты

## 🏆 Рекомендуемые варианты (от лучшего к хорошему)

### 1. **Railway.app** ⭐ ЛУЧШИЙ ВЫБОР
**Плюсы:**
- ✅ Бесплатный план: $5 кредитов в месяц (хватает для бота)
- ✅ Простой деплой через GitHub
- ✅ Автоматический перезапуск при падении
- ✅ Постоянное хранилище файлов (важно для ваших CSV/JSON)
- ✅ Логи в реальном времени
- ✅ HTTPS из коробки

**Как развернуть:**
1. Зарегистрируйтесь на [railway.app](https://railway.app) через GitHub
2. Создайте новый проект → "Deploy from GitHub repo"
3. Выберите ваш репозиторий с ботом
4. Railway автоматически определит Python
5. Добавьте переменную окружения: `TELEGRAM_BOT_TOKEN=ваш_токен`
6. В настройках проекта укажите команду запуска: `python3 bot.py`
7. Готово! Бот запустится автоматически

**Важно:** В `bot.py` можно использовать `os.getenv("TELEGRAM_BOT_TOKEN")` для безопасности.

## 🔒 Безопасность токена

**ВАЖНО:** Не коммитьте токен в Git! Используйте переменные окружения:

```python
import os
TELEGRAM_BOT_TOKEN = os.getenv("TELEGRAM_BOT_TOKEN", "ваш_токен_для_локального_теста")
```

На хостинге добавьте токен как переменную окружения в настройках проекта.

## 📝 Подготовка к деплою

1. **Создайте файл `.gitignore`** (если его нет):
```
__pycache__/
*.pyc
*.pyo
sheet_versions/
reports/
.env
```

2. **Убедитесь, что `requirements.txt` содержит:**
```
python-telegram-bot[job-queue]>=20.0
certifi>=2023.0.0
```

3. **Опционально: создайте `Procfile` для некоторых платформ:**
```
worker: python3 bot.py
```

## 🔧 Railway: бот запущен, но «данные не поступают»

### 1. Данные из Google Sheets не подтягиваются

Таблица загружается по **публичной ссылке** (экспорт в CSV). На Railway нет вашего браузера и куки Google, поэтому:

- В Google Таблице откройте **Настройки доступа** (кнопка «Поделиться»).
- Выберите **«Все, у кого есть ссылка»** → право **«Читатель»** (или «Просмотр»).
- Сохраните. Без этого запрос с серверов Railway получит 403 и данные не загрузятся.

Проверка: откройте в режиме инкогнито (выйдя из Google):

`https://docs.google.com/spreadsheets/d/1mEn564G6sfJvm96ff5XAyBVphDHWFzfZML2N6uhXyq4/export?format=csv&gid=1963801173`

Если видите CSV — с Railway тоже будет работать.

### 2. Бот не реагирует на команды в Telegram

- В Railway → проект → **Variables**: должна быть переменная `TELEGRAM_BOT_TOKEN` (без кавычек, только токен от @BotFather).
- **Logs**: посмотрите логи сервиса. Ошибки вроде «Conflict» означают, что бот запущен ещё где-то (например, локально) — везде, кроме Railway, нужно остановить.
- Убедитесь, что в Telegram пишете именно тому боту, токен которого указан в Variables.

### 3. После деплоя нет истории (/history пустая)

На Railway диск **эфемерный**: при каждом новом деплое папки `sheet_versions/` и `reports/` создаются заново. Поэтому после деплоя история появляется только после первого успешного обновления:

- Отправьте боту команду **/update** (или подождите до 30 минут до автообновления).
- После этого появятся данные для **/history_today**, **/last_update** и т.д.

### 4. Полезные команды в логах Railway

В логах при старте должны быть строки вида:

- `******** ПЕРВИЧНЫЙ ЗАПРОС ДАННЫХ ИЗ GOOGLE SHEETS ...` — бот запустился и пошёл в таблицу.
- Если сразу после этого ошибка про HTTP 403 или «Не удалось загрузить таблицу» — включите доступ по ссылке к таблице (п. 1).

## 💡 Альтернатива: ваш Mac (если он всегда включен)

Если ваш Mac всегда включен и подключён к интернету, можно запускать бота локально:

```bash
# Запуск в фоне через screen или tmux
screen -S telegram_bot
python3 bot.py
# Нажмите Ctrl+A, затем D для отсоединения
```

Или используйте `launchd` на Mac для автозапуска при загрузке системы.
