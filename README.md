# Сравнение версий таблицы «Потребность»

Скрипт загружает таблицу Google Sheets по ссылке, сохраняет текущую версию и сравнивает её с предыдущей.

## Что делает скрипт

1. **Загружает** таблицу по URL (экспорт в CSV).
2. **Сохраняет** текущую версию в папку `sheet_versions/` с именем `sheet_YYYYMMDD_HHMMSS.csv`.
3. **Сравнивает** с предыдущей сохранённой версией и формирует отчёт:
   - какие строки **добавились**;
   - какие строки **удалились**;
   - **отдельный блок** по столбцу «Количество ресурсов, необходимое к подбору»: у какой строки (ТК + тип ресурса) какое было и стало значение.

Отчёты сохраняются в папку `reports/` и **дублируются в консоль** (полный текст отчёта выводится в терминал).

## Запуск

### Mac — в одну кнопку

Если при двойном щелчке по `run.command` появляется сообщение **«повреждён»** или «не удаётся открыть»:

1. Откройте **Терминал** (Программы → Утилиты → Терминал).
2. Выполните (подставьте свой путь к папке `lika`):
   ```bash
   cd /путь/к/lika
   xattr -cr run.command
   sed -i '' 's/\r$//' run.command
   chmod +x run.command
   ```
   Или один раз запустите скрипт-исправление:
   ```bash
   cd /путь/к/lika
   bash fix_run_mac.sh
   ```
3. После этого снова дважды щёлкните по `run.command` в Finder.

В обычном случае один раз в Терминале:
```bash
cd /путь/к/lika
chmod +x run.command
```
Дальше можно запускать **двойным щелчком по `run.command`** — откроется Терминал, выполнится скрипт. В конце нажмите Enter для закрытия окна.

Через Терминал на Mac:
```bash
cd /путь/к/lika
./run.sh
# или
python3 sheet_diff.py
```

### Windows

```bash
cd c:\project\lika
python sheet_diff.py
```

Требуется Python 3.6+ (используется только стандартная библиотека).

## Доступ к таблице

Скрипт использует публичный экспорт в CSV. Чтобы он работал:

- В Google Таблице: **Настройки доступа** → «Все, у кого есть ссылка» → **Просмотр**,  
  **или**
- **Файл** → **Опубликовать в интернете** → выбрать лист и формат «CSV».

Если таблица только для своих, нужно будет использовать [Google Sheets API](https://developers.google.com/sheets/api) и учётные данные (сервисный аккаунт или OAuth).

## Структура после запуска

```
lika/
  sheet_diff.py
  sheet_versions/    # снимки таблицы (sheet_20260303_120000.csv и т.д.)
  reports/           # отчёты (report_20260303_120001.txt и т.д.)
```

Первый запуск только сохраняет версию. Сравнение и отчёт появятся со второго запуска.
